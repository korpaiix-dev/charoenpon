# 04 — บริการหลังการขาย (After-Sale Service)

> หลังลูกค้าจ่ายเงินแล้ว — เลี้ยง customer ไว้ ต่ออายุ ป้องกัน churn
>
> **6 items** · [13] SOS → [14] กลุ่มเข้าไม่ได้ → [15] renewal reminder → [16] expired → ~~[17] refund~~ → [18] tier change

(ตัด [17] refund + [19] affiliate ออกแล้ว)

---

## [13] SOS — ลูกค้าทักหาความช่วยเหลือ

### 🔍 ตอนนี้เป็นยังไง

- ลูกค้าทักว่า 'ลิ้งพัง', 'เข้ากลุ่มไม่ได้', 'หาย', 'ไม่เห็นเหี่ยจ่าย', 'หิวข้าวแล้ว' ฯลฯ
- Prae AI ตรวจจับ → ถ้าเป็น SOS → แจ้งลูกน้องในห้องสลิป + dashboard
- ลูกน้องเห็น SOS button: Resolve / Snooze / Send link
- มี SOS hybrid: ระบบช่วยตอบบางส่วน + ลูกน้องตอบที่เหลือ

### 📝 ลูกน้องเห็นอะไร

```
┌─── 🛟 SOS Console ──────────────────────────────────┐
│                                                       │
│  รายการ SOS รอจัดการ (3 อัน):                          │
│                                                       │
│  🆘 14:32 · @user789 (VIP active 5d)                 │
│    'ลิ้งหายค่ะ ส่งใหม่ให้หน่อย'                          │
│    [💡 ส่งลิ้ง VIP_1, VIP_2 (auto-detect)]            │
│    [💬 DM ลูกค้า] [✓ Resolve] [⏰ Snooze 1h]            │
│                                                       │
│  🆘 14:15 · @user456 (สมาชิกหมดวันนี้)                │
│    'ทำไมเข้ากลุ่มไม่ได้แล้วครับ'                          │
│    [💡 สมาชิกหมดอายุแล้ว — เสนอต่ออายุ]                 │
│    [💬 DM ลูกค้า] [💰 ส่งลิ้งชำระเงิน]                   │
│                                                       │
│  🆘 13:58 · @user123                                  │
│    'อิอิ' (ไม่ใช่ SOS ชัดเจน)                            │
│    [💡 ไม่ใช่ SOS — close หรือเปิด chat]               │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 🎯 ลูกน้องทำอะไรได้

- ดูรายการ SOS รอจัดการ
- ดู context (สิทธิ์เหลือกี่วัน + ประวัติ)
- ส่งลิ้งใหม่ (resend invite) — 1 click
- DM โต้ตอบลูกค้า (ผ่าน admin bot)
- Resolve / Snooze 1h-24h
- ดูประวัติ SOS ของลูกค้าคนนี้

### 🗄 DB schema

ใช้ `sos_alerts` ที่มีแล้ว — ขยาย:
```sql
ALTER TABLE sos_alerts ADD COLUMN suggested_action TEXT;  -- ระบบแนะนำ
ALTER TABLE sos_alerts ADD COLUMN resolved_by BIGINT;
ALTER TABLE sos_alerts ADD COLUMN resolved_at TIMESTAMP;
ALTER TABLE sos_alerts ADD COLUMN snooze_until TIMESTAMP;
```

### ⏱ Effort
**2 วัน** (UI + smart suggestions)

## [15] Renewal Reminder (เตือนต่ออายุ)

### 🔍 ตอนนี้เป็นยังไง

Cron job ทุกเช้า → เช็คใครจะหมดอายุใน 3 วัน → ส่ง DM

ข้อความ hardcode + variant A/B testing (ตัดออกแล้ว)

### 🗄 DB schema

`bot_messages` key: 'renewal_3d', 'renewal_1d', 'renewal_today'

```sql
CREATE TABLE renewal_reminder_config (
  trigger_days_before INT PRIMARY KEY,
  message_key VARCHAR(64) REFERENCES bot_messages(message_key),
  enabled BOOLEAN DEFAULT TRUE,
  use_promo_id INT REFERENCES promotion_campaigns(id),  -- optional
  updated_at TIMESTAMP DEFAULT NOW()
);
```

**2 วัน**
